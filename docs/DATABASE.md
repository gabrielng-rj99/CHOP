# Documentação do Banco de Dados — Licenses Manager

Este documento detalha a estrutura, os relacionamentos, os scripts e as melhores práticas para o banco de dados do Licenses Manager.

---

## Visão Geral

O banco de dados do Licenses Manager foi projetado para garantir integridade referencial, rastreabilidade e flexibilidade na gestão de licenças de software, empresas, unidades, categorias, linhas e usuários.

---

## Estrutura das Tabelas Principais

### 1. `users`
- **Campos:** id (UUID), name, email, password_hash, created_at
- **Descrição:** Usuários do sistema, responsáveis pelo acesso e operações administrativas.

### 2. `companies`
- **Campos:** id (UUID), name, registration_id (CNPJ), archived_at, created_at
- **Descrição:** Empresas clientes que possuem licenças e unidades.

### 3. `entities`
- **Campos:** id (UUID), name, client_id (FK para companies), created_at
- **Descrição:** Unidades físicas ou departamentos vinculados a empresas.

### 4. `categories`
- **Campos:** id (UUID), name
- **Descrição:** Classificação geral dos produtos licenciados.

### 5. `types` (ou `lines`)
- **Campos:** id (UUID), name, category_id (FK para categories)
- **Descrição:** Linha ou família principal do produto, vinculada a uma categoria.

### 6. `licenses`
- **Campos:** id (UUID), name, product_key, start_date, end_date, line_id (FK para types), client_id (FK para companies), entity_id (FK para entities, opcional)
- **Descrição:** Licenças de software gerenciadas pelo sistema.

---

## Relacionamentos e Constraints

- **companies → entities:** Uma empresa pode ter várias unidades.
- **companies → licenses:** Uma empresa pode ter várias licenças.
- **entities → licenses:** Uma unidade pode ter várias licenças (opcional).
- **categories → types:** Uma categoria pode ter várias linhas.
- **types → licenses:** Uma linha pode ter várias licenças.

- **Constraints Importantes:**
  - `registration_id` (CNPJ) único em companies.
  - Nomes únicos por categoria em types.
  - Não é permitido deletar uma categoria com linhas associadas.
  - Não é permitido deletar uma linha com licenças associadas.
  - Não é permitido deletar uma empresa com licenças ativas.
